# Idea 04: 真实世界版型数据集（RealPattern Benchmark）

## 概述

构建首个大规模**真实世界版型数据集**——RealPattern Benchmark，包含由专业版型师手工制作或从工业CAD系统中导出的真实版型数据。当前领域所有数据集（GarmentCodeData 130K、DressCode 20.3K、SewFactory 19.1K、GCD-MM 120K）都是程序化合成的，来自同一个GarmentCode引擎。这导致了训练分布偏移：模型学到的是"GarmentCode程序的输出分布"而非"真实版型的分布"。RealPattern Benchmark将同时提供标准化评估协议和真实分布的训练数据。

## 指向的Gap

- **Gap 5: 数据集偏差——合成 vs 真实** — 所有训练数据来自程序化合成，存在根本性分布偏移
- **Gap 27: 评估指标的碎片化** — 每篇论文用不同数据集和指标，缺乏公平比较基准

## 推导过程

### 从Gap到Idea的逻辑链

1. **核心观察**：DressWild (2026) 明确承认 "prior approaches fine-tune on synthetic garment datasets...often struggle to generalize to in-the-wild images"。GarmentDiffusion (IJCAI 2025) 指出DressCode的文本描述是 "coarse sewing pattern descriptions generated by GPT-4V"。
2. **类比CV领域**：ImageNet（真实图像数据集）的出现彻底改变了计算机视觉研究的面貌。在此之前，许多方法在合成数据上表现良好但无法泛化到真实场景。版型生成领域正处于同样的"pre-ImageNet"阶段。
3. **数据来源可行性**：多个渠道可以获取真实版型数据——开源缝纫社区（FreeSewing.org）、时尚院校的教学版型、工业合作伙伴的DXF文件、历史版型档案。

## 详细设计

### 1. 数据来源与收集策略

| 来源 | 预估规模 | 数据格式 | 获取难度 |
|------|---------|---------|---------|
| **FreeSewing.org** | 100+ patterns | 参数化JavaScript | 低（开源） |
| **Seamly2D/Valentina** | 200+ patterns | .val/.sm2d文件 | 低（开源CAD） |
| **时尚院校合作** | 500-1000 patterns | 纸质→扫描数字化 | 中（需建立合作） |
| **工业合作伙伴** | 1000-5000 patterns | DXF/Gerber/Lectra格式 | 高（商业敏感） |
| **历史版型档案** | 200-500 patterns | 纸质→扫描数字化 | 中（图书馆/博物馆） |
| **独立版型设计师** | 300-500 patterns | PDF/DXF | 中（付费收集） |

**目标规模**：首期 2000-3000 个真实版型

### 2. 数据标准化流程

每个版型样本包含：

```
{
  "id": "RP-00001",
  "source": "industrial_partner_A",
  "garment_type": "women_blouse",
  "panels": [
    {
      "name": "front_bodice",
      "geometry": [...],  // 统一为B-spline控制点
      "seam_allowance": {...},  // 各边缝份宽度
      "notches": [...],  // 标记点位置
      "grain_line": {...}  // 布纹线方向
    },
    ...
  ],
  "stitches": [...],  // 缝合关系
  "fabric_spec": {
    "type": "cotton_poplin",
    "weight": 120,  // g/m²
    "width": 150,   // cm
    "stretch": {"warp": 2, "weft": 5}  // %
  },
  "size_spec": {
    "system": "EU",
    "base_size": 38,
    "measurements": {...}
  },
  "annotations": {
    "difficulty": "intermediate",
    "construction_notes": "...",
    "text_description": "...",
    "rendered_image": "..."  // 3D仿真渲染图
  }
}
```

### 3. 标准化评估协议

**数据划分**：
- Train: 60%, Validation: 15%, Test: 25%
- Test set按复杂度分层：Simple (≤6 panels), Medium (7-12 panels), Complex (13+ panels)
- Test set按品类平衡：tops, bottoms, dresses, outerwear

**标准化指标套件**：

| 层级 | 指标 | 说明 |
|------|------|------|
| **几何级** | Panel Chamfer Distance (PCD) | panel形状精度 |
| **几何级** | Edge Curve Error (ECE) | 边曲线拟合误差 |
| **拓扑级** | Stitch Accuracy (SA) | 缝合关系正确率 |
| **拓扑级** | Panel Count Accuracy (PCA) | panel数量预测准确率 |
| **物理级** | Simulation Drape Error (SDE) | 仿真悬垂差异 |
| **物理级** | Fit Error on Body (FEB) | 在身体上的合体性偏差 |
| **感知级** | CLIP Similarity (CS) | 渲染图的视觉相似度 |
| **感知级** | Human Preference (HP) | 人类偏好评分 |
| **制造级** | Seam Allowance Correctness (SAC) | 缝份添加正确性 |
| **制造级** | Notch Completeness (NC) | 标记点完整性 |

为所有主要范式提供参考实现：

## 参考文献分析

### 核心参考

1. **GarmentCodeData** (Korosteleva et al., ECCV 2024)
   - 最大的版型数据集（130K），但完全合成
   - 其数据格式和组织方式可作为参考

2. **DressCode** (He et al., SIGGRAPH 2024)
   - 20.3K样本，文本描述由GPT-4V生成
   - GarmentDiffusion指出其文本描述质量粗糙

3. **DressWild** (Tao et al., 2026)
   - 明确承认合成数据泛化问题
   - 其使用真实照片作为输入的策略可以与RealPattern结合

4. **FreeSewing.org**
   - 开源参数化版型平台，社区贡献
   - JavaScript实现的参数化版型，可以直接提取几何数据

5. **ImageNet** (Deng et al., CVPR 2009)
   - 标准化基准改变了CV领域——类比版型生成领域的需求
   - 其数据组织、评估协议、竞赛机制可以借鉴

## 风险与挑战

1. **工业数据的商业敏感性**：品牌版型是商业秘密，获取难度大
2. **数据格式多样性**：DXF/Gerber/Lectra等格式的解析和统一需要大量工程工作
3. **标注一致性**：不同来源的版型标注规范不同，需要统一标准
4. **版权问题**：某些版型设计受版权保护，需要明确授权
5. **规模限制**：真实版型的获取成本远高于合成数据，初期规模可能有限（2-3K vs 合成130K）
