# 方向二：多模态基础模型 for 版型生成

## 概述

随着大规模多模态基础模型（如 GPT-4V、Llama-3、CLIP）的快速发展，将这些模型应用于版型生成成为一条极具前景的技术路线。其核心理念是：利用基础模型强大的视觉理解和语言理解能力，直接从图像或文本描述中推断版型结构。代表性工作包括 AIpparel（利用多模态 tokenizer 和 GPT 式架构）、ChatGarment（微调 LLM 生成 DSL 代码）、以及 DressWild（从野外图像重建版型）。然而，这一方向面临着数据基础薄弱、多层服装处理缺失、语义歧义未解、以及编辑一致性难以保证等根本性挑战。

---

## Gap 5: 数据集偏差——合成 vs 真实

### 问题描述

当前用于训练版型生成模型的所有大规模数据集都是**程序化合成**（procedurally synthesized）的，没有任何数据集包含真实版型师手工制作的版型数据。这造成了一个根本性的分布偏移（distribution shift）问题：合成数据的版型具有数学上的规整性和参数化的可预测性，而真实版型包含大量基于经验的不规则调整——非对称修正、基于试衣反馈的局部修改、面料特性驱动的补偿量、以及个体体型差异导致的特殊处理。在这种合成数据上训练的模型，其泛化到真实世界输入的能力存在根本性疑问。

### 现有方法的局限

当前主要数据集的对比分析：

| 数据集 | 规模 | 输入模态 | 来源 | 真实版型 |
|--------|------|----------|------|----------|
| DressCode (SIGGRAPH 2024) | 20.3K | 文本 | GarmentCode 程序化生成 | 否 |
| SewFactory (2024) | 19.1K | 图像 | 程序化生成 | 否 |
| GarmentCodeData (GCD) | 130K | 图像 | GarmentCode 程序化生成 | 否 |
| GCD-MM / AIpparel | 120K | 文本+图像+编辑 | GarmentCode 程序化生成 | 否 |

**关键观察**：

1. **所有数据集均来自合成源**：GarmentCode 及其变体是几乎所有数据集的生成引擎。这意味着整个领域的训练信号来自**同一套参数化模型**的输出，数据多样性受到该模型表达能力的硬性限制。
2. **文本标注质量堪忧**：DressCode 的文本描述由 GPT-4V 自动生成，GCD-MM 的文本标注由 Llama-3.1-8B 生成。这些自动生成的描述倾向于使用模糊的定性词汇（"elegant"、"flowing"），而非版型师使用的精确定量描述（"前中心线下移 2cm"、"袖山高 15cm"）。
3. **DressWild 的诚实承认**（2026）：该论文在 Introduction 中明确指出 "prior approaches fine-tune on synthetic garment datasets...often struggle to generalize to in-the-wild images"（先前方法在合成数据集上微调……通常难以泛化到野外图像）。DressWild 试图通过引入真实照片作为输入来缓解这一问题，但其训练数据**仍然是合成的**——它只是将真实图像作为测试输入，并未解决训练分布偏移的根本问题。

### 证据来源

| 来源 | 关键证据 |
|------|----------|
| DressWild (2026) | §Introduction: "prior approaches...struggle to generalize to in-the-wild images" |
| AIpparel (2025) | 120K 数据集全部由 GarmentCode 程序化生成 |
| DressCode (SIGGRAPH 2024) | 20.3K 条文本描述由 GPT-4V 生成 |
| GarmentDiffusion (IJCAI 2025) | 指出 DressCode 的 "coarse sewing pattern descriptions generated by GPT-4V" |
| SewFactory (2024) | 19.1K 样本全部程序化生成 |

### 影响程度评估

**极高（Critical）**。数据是深度学习的根基。当所有训练数据都来自同一个程序化生成引擎时，模型学到的不是"服装版型的通用规律"，而是"GarmentCode 这个特定程序的输出分布"。这带来了多重风险：

1. **过拟合到合成分布**：模型在合成测试集上表现良好，但面对真实设计师的手稿、工业版型照片、或真实消费者的描述时性能骤降。
2. **评估指标虚高**：由于训练集和测试集来自同一分布，当前报告的指标（Panel L2、Chamfer Distance 等）可能大幅高估了方法的实际可用性。
3. **创新天花板**：合成数据只能覆盖 GarmentCode 能表达的设计空间，超出该空间的设计（如不对称裁剪、解构主义、立体剪裁）在数据中完全缺席，模型无从学习。

**潜在解决路径**：构建包含真实工业版型的数据集（与版型工作室或服装院校合作），或采用 sim-to-real 迁移技术（如域随机化 domain randomization、域适应 domain adaptation）来缓解分布偏移。

## Gap 6: 多层服装的联合生成

### 问题描述

真实穿着场景中，人们几乎总是穿着多层服装：内衣 + 衬衫 + 外套、上衣 + 裤子 + 大衣、连衣裙 + 开衫等。然而，当前几乎所有版型生成方法都只处理**单件服装**——给定一张图像或一段描述，生成一件上衣或一条裤子的版型。多层服装的联合生成需要解决层间交互（inter-layer interaction）问题：不同层的服装在空间上相互重叠、在物理上相互影响（摩擦、碰撞、滑动），版型设计需要考虑这些因素（如外套需要比衬衫更大的松量 ease 以容纳内层服装）。

### 现有方法的局限

- **ChatGarment**（CVPR 2025）：所有展示的结果都是单件服装。论文未讨论多层服装的生成。
- **DressWild**（2026）：明确声明每次处理一件服装（one garment at a time），即使输入图像中人物穿着多层服装，也需要逐件分别处理。
- **NGL-Prompter**（2026）：声称支持多层服装（multi-layer），但其实现方式是**逐层独立预测**——先生成上衣版型，再独立生成裤子版型，层与层之间没有信息交互。这意味着外套的生成不知道内层衬衫的形状，无法自适应调整松量。
- **物理交互的完全缺失**：层间碰撞响应（inter-layer collision response）、面料间摩擦滑动（fabric-to-fabric friction）、重力引起的层间压迫——这些物理现象在版型设计中至关重要（决定了松量大小、衬里形状等），但在所有现有方法中**完全没有被建模**。

### 证据来源

| 来源 | 关键证据 |
|------|----------|
| ChatGarment (Bian et al., CVPR 2025) | 所有结果展示为单件服装 |
| DressWild (2026) | 明确逐件处理 |
| NGL-Prompter (2026) | 声称多层但逐层独立预测 |
| 物理模拟文献 | 层间碰撞/摩擦建模在图形学中已有研究，但未被引入版型生成 |

### 影响程度评估

**高（High）**。多层联合生成的缺失严重限制了 AI 版型生成在实际设计场景中的应用。时尚设计师和版型师在设计外套时必须考虑内层服装的空间占用，在设计套装时需要确保各件服装的比例协调。逐件独立生成的方式可能导致外套太紧（忽略内层体积）或套装各件之间比例失调。此外，虚拟试穿（virtual try-on）场景越来越需要完整的穿搭组合，单件生成无法满足这一需求。

**潜在方向**：开发层级感知的版型生成架构，其中外层服装的生成以内层服装的版型为条件输入；或引入可微分物理模拟器来建模层间交互，将物理约束作为生成过程中的约束条件。

## Gap 7: 文本描述的语义歧义

### 问题描述

文本驱动的版型生成面临一个核心语义问题：**自然语言中的服装描述词汇具有高度的歧义性和文化依赖性**。同一个词在不同上下文中可能指向完全不同的物理规格：

- **"宽松"（loose/relaxed）**：在运动休闲品牌中可能意味着比贴体多 10-15cm 的胸围松量，在高级成衣品牌中可能仅意味着 5-8cm。
- **"中长裙"（midi）**：在欧美时尚中通常指膝盖以下到小腿中部，在亚洲市场可能更偏向小腿下方到脚踝。
- **"修身"（slim fit / fitted）**：在不同品牌的尺寸系统中差异可达 5cm 以上。
- **"高腰"（high-waisted）**：相对于自然腰线上移多少因品牌和时代而异。

### 现有方法的局限

- **DressCode / SewingGPT**（SIGGRAPH 2024）：文本标注由 GPT-4V 生成，使用的是**非量化的描述性语言**（如 "a flowing maxi dress with gathered waist"）。这些描述在人类读者看来是有意义的，但在转化为精确的版型参数时存在严重的多义性。
- **GarmentDiffusion**（IJCAI 2025）：论文明确指出 DressCode 的文本标注是 "coarse sewing pattern descriptions generated by GPT-4V"（GPT-4V 生成的粗糙版型描述），因此重新设计了标注流程，使用 Llama-3.1-8B 生成更精细的 brief + detailed 双层描述。但即使是改进后的标注，仍然缺乏与具体数值测量的锚定（grounding）——没有将 "slightly above the knee" 映射到具体的裙长数值。
- **ChatGarment**（CVPR 2025）：支持自然语言交互，但将语义解析完全委托给 LLM。LLM 的解析结果取决于其训练数据中的文化偏见，无法保证跨文化的一致性。
- **定量标准的缺失**：在工业版型制作中，每个描述词汇都有**品牌内部的量化定义**（如 "regular fit" = 胸围松量 12cm, "slim fit" = 胸围松量 6cm），并通过尺寸表（size chart/grade rule）明确规定。当前没有任何 AI 方法引入这种量化标准体系。

### 证据来源

| 来源 | 关键证据 |
|------|----------|
| GarmentDiffusion (IJCAI 2025) | 明确指出 DressCode "coarse sewing pattern descriptions generated by GPT-4V" |
| GarmentDiffusion (IJCAI 2025) | 重新设计标注流程但仍缺乏数值锚定 |
| ChatGarment (CVPR 2025) | 语义解析完全依赖 LLM，无文化适配机制 |
| 工业实践 | 品牌内部有量化的松量/比例定义，AI 方法未引入 |

### 影响程度评估

**中高（Medium-High）**。语义歧义的影响在学术研究的评估框架内可能不明显（因为训练集和测试集的标注来自同一分布），但在实际应用中会导致严重问题。当真实用户输入 "我想要一件宽松的长款大衣" 时，AI 生成的版型可能与用户期望相差甚远——因为 "宽松" 和 "长款" 在不同用户心中的数值映射截然不同。

**潜在解决路径**：

1. **引入量化标准层**：建立从描述词汇到数值范围的映射表（可按品牌/地域/风格分类），将模糊的自然语言转化为有界的参数范围。
2. **交互式消歧**：当检测到歧义描述时，通过追问引导用户明确意图（如 "您希望宽松到什么程度？A.运动休闲风 B.通勤商务风 C.宽大oversize风"）。
3. **参考图像锚定**：允许用户提供参考图像来消除文本描述的歧义，利用多模态融合来精确化语义。

## Gap 8: 版型编辑的语义一致性

### 问题描述

版型设计是一个高度迭代的过程——设计师很少一次就生成满意的版型，而是需要反复调整、修改。AIpparel 和 ChatGarment 都支持基于文本指令的版型编辑（如 "make sleeves longer"、"add darts to the bodice"），但当前的编辑操作缺乏**全局一致性保证**。在真实版型制作中，修改一个面片通常需要级联修改多个关联面片——延长袖子需要同步调整袖窿弧线，放大胸围需要调整前后片和侧缝，改变裙长可能需要调整腰围比例。这种全局约束传播（global constraint propagation）在当前方法中完全缺失。

### 现有方法的局限

- **ChatGarment**（CVPR 2025）：论文在局限性部分明确承认 "altering the skirt length might unintentionally affect other parts of the garment"（修改裙长可能无意中影响服装的其他部分）。这说明其编辑操作是**局部的**（只修改被指定的参数），没有考虑参数之间的全局依赖关系。
- **AIpparel**（2025）：支持通过 token 级别的编辑来修改版型，但编辑操作的粒度是面片级别的——修改一个面片的 token 不会自动触发关联面片的调整。论文未讨论编辑一致性问题。
- **版本管理的缺失**：在传统版型设计软件（如 Lectra Modaris）中，修改历史可以追溯和撤销。当前所有 AI 方法都没有 "undo/redo" 或版本管理概念——一旦编辑结果不满意，用户无法回退到上一个版本，只能重新开始或再次编辑。
- **参数化约束 vs 自由形变**：GarmentCode 系列由于其参数化本质，某些编辑可以通过调整参数实现（如袖长参数），约束关系由程序逻辑保证。但对于超出参数化范围的自由编辑（如手动修改面片形状），则完全没有约束传播机制。

### 证据来源

| 来源 | 关键证据 |
|------|----------|
| ChatGarment (Bian et al., CVPR 2025) | §Limitations: "altering the skirt length might unintentionally affect other parts" |
| AIpparel (2025) | 支持 token 级编辑但无跨面片约束传播 |
| 工业 CAD 系统 | Lectra Modaris 等提供参数联动和修改历史，AI 方法均不具备 |

### 影响程度评估

**高（High）**。编辑一致性直接影响用户体验和实际可用性。在交互式设计场景中，用户期望 "make sleeves longer" 这样的指令能产生协调一致的全局修改，而不是仅仅延长袖片而导致袖窿不匹配。当前方法的局部编辑行为可能导致：

1. **几何不一致**：编辑后的面片之间缝合边长度不匹配，导致物理模拟时出现褶皱或拉伸。
2. **比例失调**：局部修改破坏了服装整体的视觉比例关系。
3. **累积误差**：多轮编辑后，每一步的局部不一致累积放大，导致最终版型严重偏离预期。

**潜在解决路径**：

1. **约束图传播**：显式建模面片参数之间的依赖关系图，编辑一个参数时自动沿图传播修改。
2. **编辑感知的生成模型**：训练模型同时预测编辑目标面片和所有需要联动修改的面片。
3. **编辑历史栈**：维护编辑操作的序列化历史，支持撤销/重做和分支编辑。

## 总结：方向二研究差距全景

| Gap | 核心问题 | 影响程度 | 短期可解性 |
|-----|----------|----------|------------|
| Gap 5: 数据集偏差 | 全部合成数据，无真实版型 | 极高 | 低（需要工业合作获取真实数据） |
| Gap 6: 多层服装联合生成 | 逐件独立，忽略层间交互 | 高 | 中（可引入条件生成机制） |
| Gap 7: 文本语义歧义 | 描述词汇无量化锚定 | 中高 | 中（可建立量化标准层） |
| Gap 8: 编辑语义一致性 | 局部编辑破坏全局一致 | 高 | 中（可引入约束传播机制） |

**核心洞察**：方向二的四个差距揭示了一个更深层的问题——**当前的多模态基础模型方法在"理解服装"和"生成版型"之间缺少一个关键的中间层：服装设计知识的结构化表示。** 基础模型擅长从文本/图像中提取外观特征，但不具备版型师的领域知识（如松量概念、缝合约束、比例关系、层间适配）。数据集的合成偏差（Gap 5）使模型无法从数据中自动学到这些知识；多层处理的缺失（Gap 6）和编辑一致性问题（Gap 8）进一步暴露了模型对服装结构理解的浅层性。要突破这些瓶颈，需要将**服装设计的领域知识显式编码**到模型架构或训练过程中——无论是通过知识图谱、结构化约束、还是专家引导的数据标注。
